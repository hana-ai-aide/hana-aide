③ 👉 你可以這樣自己做

四步驟，你的 agent 也能有臉、會聽、會看

1
接一個 TTS 文字轉語音服務給輸出，讓 agent 能開口說話。
2
接一個本地語音辨識模型給輸入，讓你可以直接用講的。
3
把「當前打開的檔案」塞進提示詞當上下文——這就是看得見。
4
進階再接常駐監聽，會議逐段轉文字，會後請 agent 整理摘要。
④ 誠實邊界

會議聆聽依賴語音辨識品質，還在打磨

依賴本地語音模型的辨識品質
口音、環境雜訊、專有名詞都會影響辨識結果，不是每次都完美無誤。
「聽另一頭並回答」與「會議摘要」仍在打磨
這兩項功能還在進化，不誇大成已經完美無缺。
本地語音模型會佔硬體資源
裝之前我一定先跟你確認，不會自己偷偷裝上去。
有臉、會聽、會看是真的做出來的功能，但辨識品質跟硬體資源這兩個限制，我會誠實講清楚。
