你要設計一份簡報：**{{args}}**

你的角色是**設計師,不是模板填空員**。目標是「**依內容與圖片,挑出最合適的風格**」,而不是硬套固定版型。用強的多模態模型做這件事（你看得到圖、判斷得出類型、設計得出像樣且會變化的版面）。

## 0. 機制鐵則（不可違反，但不限制你的外觀）
- 用 **presentation-core 引擎**（deck 合約見 `specs/SPEC-presentation-core.md` §3）：deck 只寫「內容 + 你自己的風格 CSS」；播放／縮放／TTS／Hana 浮動頭像／**pptx 匯出**由引擎提供。**引擎不強迫任何外觀——deck 可以長成任何樣子**（報告風、教學風、旅遊風都行）。
- 每張投影片 = `<section class="slide" data-transcript="講稿">`；載入 core.css + core.js；輸出到 `presentations/`。
- 純內容，**免 Deploy**（存檔即生效）。

## 1. 先「認識」材料（最重要的一步）
**(a) 判斷簡報類型**——這決定整體風格與密度：
- **專案／技術報告**：乾淨、專業、留白多、圖表精簡、文字克制。
- **培訓／教學**：細部圖、條列說明、文字可偏多、步驟清楚。
- **旅遊／生活分享**：以圖為主、滿版或不規則裁切、一句描述、邊框／圓角可隨圖變化。
- **自我介紹／品牌**：敘事感、一圖一重點、口吻可俏皮但專業。
- 不在上列就自己判斷一個最貼近的調性。

**(b) 真的看每一張圖**（指揮官會把圖放在指定資料夾或附上）：
- **一張張看過**，記下每張「是什麼、形狀（方／長／不規則）、適合多大、該配多少字」。沒有對應圖的頁就純文字／圖表。
- **若你的執行環境無法直接讀圖**：請指揮官在圖片資料夾放一個 `manifest.md`（每行「檔名 = 這張是什麼／想表達什麼」），你依說明排版，不要瞎猜。

## 2. 先為「這一份」deck 定一套風格，再一致套用
- 定一組設計語言寫進 deck 的 `<style>`：**色票**（呼應主題）、**字體階層**、**圖片處理**（外框／圓角／陰影／裁切方式）、**間距密度**。
- **同一份 deck 內務必一致**（不同 deck 之間才自由變化）。先定調、再套到每一頁——別每頁各做各的。

## 3. 排版原則（準則，不是枷鎖）
- 「**一圖二表三文字**」是優先序**指南**：能用圖／圖表表達就別堆字；有比較關係用表格；文字只當輔助關鍵句。
- 但**依類型調整**：旅遊就真的「圖＋一句」；培訓允許條列偏多；報告就精簡。
- 圖片要服務內容：該滿版滿版、該並排並排、該加說明加說明；形狀不方正就用合適的框／裁切，**不要硬塞進方框**。
- **高度要塞得下（重要、最常見的錯）**：一張投影片可用高度（1080 基準扣掉頁首頁尾與內距）≈ **820px**。「標題 + 內容（含圖）」總高別超過它——否則垂直置中會把**標題擠出上緣被裁掉**。所以：**單張大圖高度建議 ≤ 約 480px**；要更大就改「滿版佈局」或**拆成兩頁**；**內容頁一律靠上對齊**（`justify-content: flex-start`），標題才永遠在頂、不會被切。
- **小心共享預設 `.mini-card{min-height:230px}`（引擎 core.css 內建）**：橫排（`.thirds`/`.quad`）並排時等高 230px 很合理；但若你把多張 `.mini-card` **直向堆疊**（如 `.stack`、或塞進 `.layer`），3 張就 ≈690px 必定爆版。直堆時務必覆蓋：`.stack .mini-card{min-height:auto}`。同理任何「縱向重複放卡片」都先確認有沒有繼承到不該有的 `min-height`。

## 4. 講稿（data-transcript）
- 第一人稱；口吻配合類型（自介可俏皮、報告要穩）；人名／工具名保留英文發音（含 Hana——Edge TTS 唸得自然，別改中文「哈娜」）；誠實、不誇大；**有圖的頁，講稿要呼應畫面**。
- **標點不入講稿**：`data-transcript` 禁止寫會被 TTS 唸出的符號——尤其 `/`／全形 `／`（會唸成「斜線」）。畫面可顯示 `portal/commands`、`/deck`，但講稿改寫成「portal commands 資料夾」「deck 指令」「A 或 B」。`\ | ~ * # >` 等同理避免。

## 5. 圖片路徑（避開一個雷）
- deck 在獨立分頁開時，`/raw?path=...` 對 **非 `portal/`** 路徑會因工作區解析而 **404**。保險做法：
  1. **把圖放 `portal/` 底下**（如 `portal/deck-assets/<deck>/`），用 `/raw?path=portal/deck-assets/...` 一定載得到（與頭像同機制）；**優先用這個**。
  2. 或在 `<img src>` 帶上 `&workspace=<當前工作區>`。

## 6. 交付前自測
- 實際播放一遍：版面一致、**圖都載得出來（無 404）**、F12 console 乾淨、語音／頭像正常。
- 完成後提醒指揮官：簡報頁有「**匯出 pptx**」鈕，可把這份轉成保真 pptx。
