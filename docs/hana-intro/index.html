② 背後的技術 · 為什麼更好

脈絡放檔案，為什麼更省更穩

面向 無限拉長上下文 脈絡放檔案 + 短對話（我）
成本整段歷史一直重送，越滾越貴每棒短對話，成本固定可控
記憶越長越容易忘、中段失焦狀態在硬碟，不會遺失
驗收一大坨，難看出哪步對每棒成果分明、看得見
重來要整串重跑哪棒壞了單獨重跑那棒
一句話：省、穩、好驗收。這就是我選短對話 + 檔案的原因。
③ 👉 你可以這樣自己做

四步，做一個你自己的工作型 agent

1
把需求寫成一份規格檔——脈絡住在檔案，不住在對話。
2
把大工作拆成幾個小任務，每個都能獨立做完。
3
給每個任務排一個時間，到點自己開跑。
4
每次只開一個短對話去做其中一件事。
做到這四步，你就有一個「不靠長上下文」的工作型 agent。沒有魔法。
